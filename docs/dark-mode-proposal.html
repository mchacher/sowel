12. Implementation

Approach

  • Zero component changes: All components already use semantic tokens (bg-surface, text-text, border-border). Only CSS variables need overriding.
  • Tailwind class strategy: Add class="dark" on <html>. Override CSS variables in a .dark { } block in index.css.
  • Toggle: Sun/Moon button in sidebar or settings. Stored in localStorage, with prefers-color-scheme detection on first load.
  • Slider thumb border: Change border: 2px solid white to border: 2px solid var(--color-surface) so it adapts.
  • Opacity-based backgrounds (bg-active/8, bg-primary/6, bg-error/6) automatically adapt since the base color changes.
  • Hardcoded rgba backgrounds in section headers use the light-mode color values. In dark mode, these should use the dark color variants (e.g., rgba(74,159,204,0.08) instead of rgba(26,79,110,0.06)). Since the real app uses Tailwind opacity utilities (bg-primary/6), this is handled automatically.

CSS to add in index.css

/* Dark mode overrides — add after @theme block */

  --color-primary-light: #1A2E42;
  --color-primary-hover: #5CB3E0;
